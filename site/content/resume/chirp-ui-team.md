## Summary

We design and maintain chirp-ui — a component library for Python web apps that
renders semantic HTML on the server and enhances it with htmx and Alpine only
where a screen needs interactivity.

## Selected work

- **Component library** — 100+ Kida macros for layout, forms, navigation,
  overlays, and data display, validated against a variant/size registry.
- **chirp-theme** — the documentation theme that ships with chirp-ui, used to
  build this very site.
- **Patterns** — recipes for app shells, marketing pages, and fragment-swap
  islands that stay legible without a client framework.

## Principles

A Python vocabulary for UI, not a string vocabulary. Render first, enhance
second. Make the common screen consistent so docs and product never drift apart.
