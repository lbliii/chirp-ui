Getting Started

Introduction to chirp-ui

chirp-ui is an optional companion UI layer for the Chirp web framework — a set of Kida template macros that render as semantic HTML and are styled with modern CSS: custom properties, color-mix(), :has(), and container queries. No build step required.

The library is designed to give you a functional, accessible, token-driven component vocabulary out of the box, with sensible defaults that you can redress entirely through CSS custom properties. Everything below — the topbar, sidebar, breadcrumbs, callouts, code blocks, and even this page's prose styling — is rendered with the same chirpui-* classes you'd use in your own app.

ℹ New to the Bengal ecosystem?
chirp-ui sits on top of kida-templates (the template engine) and bengal-chirp (the web framework). You can use it without either — the compiled CSS stands alone — but the macros unlock the real power.

Install

chirp-ui is published on PyPI. Most users add it as a dependency alongside Chirp: 

uv add chirp-ui
# or
pip install chirp-ui

Register it with your Chirp app and the component loader, CSS, and JavaScript helpers are wired up automatically: 

from chirp import Chirp
from chirp_ui import use_chirp_ui

app = Chirp(__name__)
use_chirp_ui(app)  # registers loader, filters, static path, Alpine

Your first page

A chirp-ui page is just a Kida template that composes macros. Start with the app shell — it provides a sticky topbar, a sidebar, and a main content region with sensible defaults for overflow, focus, and dark-mode: 

{% from "chirpui/app_shell.html" import app_shell %}
{% from "chirpui/hero.html" import hero %}

{% call app_shell(brand="Acme") %}
  {% call hero(title="Welcome", lede="Get started in 60 seconds.") %}
    {{ btn("Read the docs", href="/docs", variant="primary") }}
  {% end %}
{% end %}

Every component accepts variants and sizes as keyword arguments, validated against a central registry. Unknown values fall back to the default with a warning — or raise in strict mode.

Variants & sizes

The validation registry is the single source of truth. You can inspect it from any shell: 

>>> from chirp_ui.validation import VARIANT_REGISTRY
>>> VARIANT_REGISTRY["button"]
frozenset({'default', 'primary', 'ghost', 'danger', 'success'})
! Strict mode in CI
Call set_strict(True) in your test harness to escalate validation warnings to ValueError. This catches typos like variant="primery" before they ship.

Theming & tokens

All colors, spacing, typography, motion, and elevation are exposed as CSS custom properties prefixed with --chirpui-. Override them on :root, on a scoped element, or via data-theme:

TokenPurposeDefault
--chirpui-accentBrand / link colorhsl(212 100% 48%)
--chirpui-surfaceCard / panel bglight-dark(#fff, #111)
--chirpui-radiusDefault corner radius0.5rem
--chirpui-durationDefault motion duration180ms
✓ Dark mode comes free
Every token uses light-dark() under the hood. Toggle <html data-theme="dark"> and the entire page reflows — try the segmented control in the top-right.

Keyboard shortcuts

Press ⌘ + K to open search, G then H to jump home, or ? anywhere to show all shortcuts. These are wired up in the default app shell and can be disabled per-page.
