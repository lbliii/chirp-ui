# Agent Source Map

**Status:** active Sprint 6 source contract
**Created:** 2026-05-12
**Plan:** `docs/plans/done/PLAN-skeleton-equivalent-roadmap.md` Sprint 6

This map connects generated agent-facing or machine-facing site outputs to the
Chirp-owned source inputs that should inform them. It does not introduce a new
generated artifact. Bengal remains the owner of site-wide generated outputs.

## Ownership Rules

- Bengal owns site-wide generated artifacts: `llms.txt`, `agent.json`, search
  indexes, page JSON, page Markdown, sitemap, and robots output.
- Chirp UI owns registry projections, durable docs, source site content, and the
  standalone `chirpui.manifest.json` emitted into the published site.
- Chirp UI may enrich source inputs consumed by Bengal, but must not replace
  Bengal-owned output names.
- A future Chirp-owned agent artifact requires a separate stop-and-ask decision,
  a distinct name, and tests that prove it does not overlap Bengal-owned files.

## Current Artifact Decision

No new Chirp-owned agent artifact is needed for Sprint 6. The correct next work
is source enrichment and provenance testing:

1. Keep Bengal-owned outputs generated by Bengal.
2. Keep `site/public/chirpui.manifest.json` as the only Chirp-owned published
   machine artifact in this phase.
3. Use `docs/AGENT-SOURCE-INVENTORY.md` to decide which examples may become
   curated snippets later.
4. Reopen a new artifact only when a concrete consumer needs data that Bengal
   outputs and the Chirp manifest cannot provide.

## Forbidden Overlaps

Chirp UI must not add build tasks or scripts that write these names directly:

- `llms.txt`
- `llms-full.txt`
- `agent.json`
- `index.json`
- `search-index.json`
- `sitemap.xml`
- `robots.txt`

If a future slice proposes a distinct artifact such as `chirpui.agent.json`, it
must document the schema, consumers, build command, collision policy, and
deprecation behavior before implementation.
